# 延时摄影自动化工具 PRD

> 版本 1.0 · 2026-06-07 · 本文档描述产品定位、目标用户、功能需求、技术实现与路线规划。

---

## 1. 产品概述

**一句话**：面向摄影师的 macOS 桌面工具，把「延时摄影 RAW 序列 → 调色 → 渲染成片 → 社交媒体多版本输出」这条繁琐链路自动化，并内置运镜、主体识别、稳定等增强。

**形态**：Electron 桌面应用（前端）+ 本地 Python(FastAPI) 后端 + Swift/AVFoundation/Vision 原生工具，驱动 Adobe After Effects / Bridge / LRTimelapse / Photoshop。

**设计原则**：
- 全本地运行，不依赖云、不装 ffmpeg（缩放/转码/分析全用 macOS 原生 AVFoundation / Vision / ImageIO / qlmanage）。
- 极简 + 液态玻璃 + 扁平化 UI，功能分区清晰。
- 自动化优先：能自动识别的（分辨率、主体、最佳帧）就不让用户手填。

---

## 2. 目标用户与场景

- **用户**：拍延时的摄影师/创作者，素材是相机 RAW 序列（如 Sony `.ARW`），成片要发抖音/小红书/B 站等。
- **核心场景**：拍完一组延时 RAW → 想要一条高质量 ProRes 母版存档 + 一条适配某平台的社媒版（竖屏/横屏、带运镜）→ 一键化完成，少在多个软件间手动倒腾。
- **次要场景**：手里已有一条成片（mov），想快速转成另一个社媒版本。

---

## 3. 核心工作流（流水线）

四阶段状态机：`BR → LRT → AE → 导出`，手动阶段会暂停等用户操作。

| 阶段 | 类型 | 做什么 |
|------|------|--------|
| **BR** | 手动 | 自动打开 Adobe Bridge 指向素材文件夹，用户在 Camera Raw 调透视/镜头配置/色差，完成点「继续」 |
| **LRT** | 手动 | 自动打开 LRTimelapse，用户做关键帧/圣光(Holy Grail)/去闪/自动过渡并保存 XMP，完成点「继续」 |
| **AE** | 自动 | AE 直接读 RAW 序列、按原始分辨率建合成、（可选增稳）、分块渲染成 ProRes 4444 母版 |
| **导出** | 自动 | 保留 ProRes 母版 + 用 AVFoundation 转出社媒版 |

- 支持**自定义工作流**（勾选只跑某些阶段）。
- 跑完/失败后有「**重新开始（换素材）**」按钮，重置流程、清素材，准备下一组。

## 4. 功能需求

### 4.1 素材与信息（自动识别）
- 选择 RAW 文件夹即可，**无需手填相机机型/分辨率**。
- 自动读取首帧 EXIF（ImageIO），展示**相机 / 镜头 / 分辨率 / 快门 / 光圈 / ISO / 焦距**。
- 母版**按 RAW 原始尺寸**建（自动适配 S35 裁切模式 / 全画幅，绝不放大虚化）。
- 序列预览：缩略图条 + 播放轮播。

### 4.2 AE 渲染（稳健 + 可视进度）
- AE 直接导入 RAW 序列（保留 LRT 的 XMP 调色），按原始分辨率、不裁切建合成。
- **分块渲染**：每段单独跑一个 aerender 进程，防止一次性渲长序列崩溃；段失败自动重试；断点续渲。
- 渲完后 Swift **无损拼接**成完整 ProRes 母版，并自动清理分块临时片段。
- **异步执行 + 帧级进度条**：点「继续」HTTP 秒回，渲染后台跑；前端每秒轮询，进度条平滑推进，显示「渲染第 N/M 段 · 第 X/总 帧」。

### 4.3 增稳（变形稳定器）⚠️ 未完成
- UI 可开关 AE 变形稳定器，配置结果（平滑运动/无运动）、平滑度、方法（位置/位置缩放旋转/透视/子空间）。
- **现状**：参数注入已修正（matchName 已实机确认）；但 Warp Stabilizer 的**运动分析未能在 aerender 渲染前完成**，输出仍带「在后台分析」横幅、不稳定。**待修**：脚本需触发分析并等待完成再渲染。

### 4.4 成片导出（母版 + 社媒版）
- **母版**：自动保留 ProRes 4444（`<素材名>_master.mov`）。
- **社媒版**（自由组合，AVFoundation 转码）：
  - 格式：H.265 / H.264
  - 画幅：16:9 横 / 9:16 竖 / 3:4 竖 / 1:1 方 / 3:2 原始（不裁），非原始画幅中心裁切
  - 分辨率：720p / 1080p / 4K
  - 文件名：`<素材名>_social_<宽>x<高>_<格式>.mp4`
- 画幅与运镜联动（竖屏横扫只在竖屏画幅出现）。

### 4.5 自动运镜
- 类型：**Ken Burns 推拉 / 平移 Pan / 竖屏横扫**；方向、强度可调。
- 实现：裁切框随时间 ramp（起框→止框），无运镜时退化为静止、零风险。
- **手动选区**：选 Ken Burns 时可弹窗在素材底图上**拖框**指定放大终点（从全貌推近到框选区域）。
- **主体识别**：可开「自动对准主体」，用 Vision 显著性检测主体，裁切/运镜锚点对准主体（锚点优先级：手动框 > 主体 > 画面中心）。

### 4.6 成片转社媒（独立 Tab）
- 导入任意已有成片（mov/mp4），套用完整社媒配置（格式/画幅/分辨率/运镜/选区/主体），直接出社媒版到源同目录。
- 与流水线**共用**同一套转码逻辑（`transcode_social`）与选区窗口（`window.cropModal`）。

### 4.7 动态背景（液态玻璃增强）
- 选素材后，自动挑该序列**饱和度最高的一帧**（CoreImage 算平均饱和度），做模糊背景垫在玻璃卡片下，让 UI「透出」你的片子。

### 4.8 UI
- 顶部 Tab：延时流水线 / 照片筛选（占位）/ 成片转社媒。
- 流水线表单分 4 个玻璃卡片：**素材与流程 / 渲染参数 / 成片导出 / 执行**。
- 进度条、阶段看板、操作引导、重新开始按钮。

## 5. 技术架构

```
Electron(前端 UI) ──HTTP/轮询──▶ FastAPI(本地后端, 端口 8756)
                                     │
                  ┌──────────────────┼─────────────────────┐
            驱动 Adobe              编排                  原生工具
        Bridge / LRT / AE     PipelineRunner          Swift:
        (AppleScript          (异步线程 + 进度)        · media_export(AVFoundation 转码/裁切/运镜 ramp)
         DoScriptFile)                                · --saliency(Vision 主体)
        aerender(分块渲染)                            · --meta(ImageIO EXIF)
                                                      · mov_concat(无损拼接)
                                                      qlmanage(缩略图)
```

- **后端模块**：`pipeline/{runner, stages, ae, export, export_formats, preview, effects, workflows, launcher, models}`。
- **裁切/像素/运镜数学**全是 Python 纯函数，可单测；Swift 只执行；外部命令注入 `run`/`stream` 便于测试。
- **测试**：pytest（后端 140+ 用例）+ Jest/jsdom（前端 27 用例），TDD 驱动。

## 6. 非功能需求

- 平台：macOS（依赖 AE 2026 / Bridge / LRTimelapse / 系统 Vision/AVFoundation/ImageIO）。
- 无云、无第三方转码依赖（不装 ffmpeg）。
- 渲染异步、不阻塞 UI；长任务有进度反馈。
- 稳健性：AE 分块防崩 + 重试 + 断点续渲 + moov 封口校验。

## 8. 关键决策记录（why）

- **AVFoundation 替代 Premiere**：原 Premiere `.epr` 方案从未跑通；母版本就是 ProRes，社媒转码用原生 AVFoundation 更可靠、无新依赖。
- **母版用 RAW 原始分辨率**：避免 S35 素材被放大虚化，也省去手选分辨率。
- **数学放 Python / 执行放 Swift**：保证核心逻辑可单测，外部调用可注入。
- **aerender OM 模板名必须用 AE 真实存在的「ProRes 4444」**（不带 Apple），否则渲不出 —— 这是早期最大的坑。

# 产品需求文档（PRD）：延时摄影自动化工具

> 版本 v1.0 · 2026-06-03 · 负责人：feitong
> 配套设计/计划见 `docs/superpowers/specs` 与 `docs/superpowers/plans`

---

## 1. 背景与问题

延时摄影后期是一条繁琐、重复、跨多个专业软件的流水线：Adobe Bridge（Camera Raw）→ LRTimelapse → After Effects → Premiere Pro（业内简称 BR-LRT-AE-PR）。摄影师每做一条片子，都要在四个软件之间反复手动切换、重复一样的导入/渲染/导出操作，耗时且容易出错。

其中真正需要**人工创意判断**的只有两步——在 Camera Raw 里做透视/镜头/色差校正，在 LRTimelapse 里设关键帧/去闪/圣光过渡。其余（渲染、增稳、转码导出）都是机械劳动，完全可以自动化。

**本产品目标**：把这条流水线里能自动化的部分自动化，把人工只留在该判断的地方，并辅以预览、模板等提效功能。

---

## 2. 目标用户

- 个人/独立延时摄影师（主力画像：Sony A7R IV 等高像素全画幅用户）
- 已有完整 Adobe 全家桶 + LRTimelapse 的进阶玩家
- macOS 用户（首版仅 macOS）

---

## 3. 产品目标与非目标

### 目标
- 一键驱动 BR-LRT-AE-PR 流水线，自动完成可自动化的阶段
- 人工只在 BR、LRT 两个创意判断点介入，工具负责"开门"（自动打开对应软件并定位素材）
- 支持不同工作流（不是所有人都走完整四阶段）
- 出片前能预览素材

### 非目标（首版不做）
- Windows 版
- 替代 LRTimelapse / 重新实现其关键帧/去闪算法
- 拍摄前的气象/光线预测
- 云端/协作/账号体系

## 4. 范围与功能

### 4.1 延时流水线（核心）

四个阶段，两个手动暂停点 + 两个自动段：

| 阶段 | 软件 | 类型 | 说明 |
|------|------|------|------|
| **BR** | Adobe Bridge + Camera Raw | 手动暂停 | 工具自动打开 Bridge 指向 RAW 文件夹；用户全选 + ⌘R 进 ACR 调透视/镜头配置/消色差，完成点「继续」 |
| **LRT** | LRTimelapse | 手动暂停 | 工具自动打开 LRTimelapse；用户做关键帧 /（日转夜走圣光 Holy Grail）/ 去闪 / 自动过渡 / 导出图像序列，完成点「继续」 |
| **AE** | After Effects | 自动 | 导入图像序列 → 变形稳定器增稳 → 渲染为 ProRes 4444 中间视频 |
| **PR** | Premiere Pro | 自动 | 导入中间视频 → 按导出格式出片 |

**关键设计依据**：LRTimelapse 6 无命令行/脚本接口，关键帧/去闪/圣光全是 GUI 操作，无法可靠自动化 → LRT 全程手动。Camera Raw 的校正每次拍摄（镜头/焦段）都不同，无法套统一预设 → BR 也手动。因此这两步设计为"工具开门 + 人工操作 + 继续"。增稳放在 AE（变形稳定器），去闪由 LRT 完成（AE 默认无可用去闪效果）。

**状态机**：BR（暂停）→ 继续 → LRT（暂停）→ 继续 → AE（自动）→ PR（自动）→ 完成；任一自动阶段失败则标记失败阶段与错误。

### 4.2 工作流模板

并非所有人都走完整四阶段。工作流 = `[BR,LRT,AE,PR]` 的有序子集。

- **内置模板**：全流程 BR-LRT-AE-PR / 跳过BR LRT-AE-PR / 无PR BR-LRT-AE / 极简 LRT-AE
- **自建模板**：勾选所需阶段、命名保存，持久化复用
- **约束**：含 PR 必含 AE（PR 消费 AE 的中间视频）

### 4.3 导出格式

- **预设模式**：母版 ProRes 422 HQ / 母版 ProRes 4444 / 交付 H.265 10bit / 社媒 H.264 高质量 / 社媒 H.264 压缩
- **手动模式**：编码 ProRes/H.264/H.265 → 对应细项（ProRes 档位；H.264 码率+质量档；H.265 码率+位深）
- **分辨率**：按相机机型动态给可选项（原分辨率/8K/4K/…），机型可自定义添加（A7R IV 等预置）
- **帧率**：常用预设 + 自由输入（1–120 fps）
- **增稳**：变形稳定器，可调 结果（平滑运动/无运动）/ 平滑度 / 方法（位置/位置缩放旋转/透视/子空间变形）

### 4.4 序列预览

- 选中 RAW 或 LRT 导出文件夹后，右侧面板显示首/中/尾缩略图 + 帧数
- 「播放」抽样帧轮播，模拟延时成片动态
- 支持 RAW(.ARW) 与 JPG/TIF/PNG（macOS QuickLook 抽缩略图）

### 4.5 易用性

- 路径输入用原生文件夹选择对话框（不手敲路径）
- 手动阶段显示完整操作引导（含圣光等清单）
- 跟随系统的暗色模式
- 极简·液态玻璃·扁平化视觉风格
- 桌面双击启动（.command）

### 4.6 AI 照片筛选（规划中，非延时）

从一批单张/人像照片里自动挑最佳时刻/构图/瞬间（清晰度、人脸质量、构图、综合美学评分）。已在总设计中规划，优先级低于延时流水线，暂缓实现。

## 5. 技术架构

- **前端**：Electron（macOS 桌面），原生 JS + HTML/CSS，液态玻璃风格
- **后端**：Python 3.9 + FastAPI（本地 127.0.0.1:8756），Electron 主进程自动拉起
- **通信**：HTTP（请求/响应）+ WebSocket（进度，预留）
- **软件驱动**：
  - AE：ExtendScript 建工程 + `aerender` 命令行渲染
  - PR：ExtendScript 导入 + 按 .epr 预设导出
  - BR/LRT：`open -a` 启动并定位文件夹
  - 预览缩略图：`qlmanage`（QuickLook）
- **持久化**：`cameras.json`（机型）、`workflows.json`（自定义工作流）
- **测试**：后端 pytest，前端 Jest/jsdom；外部软件调用以依赖注入隔离，可单测编排逻辑

## 8. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| LRTimelapse 无法自动化 | LRT 只能手动 | 设计为手动暂停 + 工具开门（已采纳） |
| Adobe 脚本 API 版本差异 | AE/PR 脚本需按机调 | 关键 matchName/模板/预设集中为常量，留容错与真机迭代任务 |
| 变形稳定器需"分析"才生效 | 增稳可能不生效 | M3 真机重点验证并补分析触发 |
| 各软件升级导致脚本失效 | 维护成本 | 常量集中、单测覆盖编排逻辑、降级为手动提示 |
| 仅 macOS | 用户范围受限 | 首版聚焦 mac；架构未绑死平台，后续可评估 |

## 9. 验收标准（M3 端到端）

- 选定一组真实 RAW/序列，选「全流程」或任一模板
- BR、LRT 两步工具能自动打开对应软件并显示引导，用户操作后「继续」可推进
- AE 自动渲出带增稳的 ProRes 4444 中间视频
- PR 自动按所选格式导出最终成片，文件可正常播放、规格符合所选（分辨率/帧率/编码）
- 全程进度看板正确反映各阶段状态，失败有明确提示

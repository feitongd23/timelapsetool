# 火烧云领域知识基线(专家视角)

> 用途:这是 skyfire 预测引擎的**领域地基**——量化打分模型、卫星云图判读、经验库(RAG)笔记格式都从这里派生。
> 立场:以资深地理气候 + 风光气象视角写,交叉验证多源(NOAA Corfidi、sunsethue、sunsetbot 公开方法论、中国气象局、科普中国、开源五因子模型)。
> 日期:2026-07-05。关联现有诊断:规则模型首测 Spearman ρ=0.026(见文末"对现有模型的批判")。

---

## 0. 一句话本质

火烧云 = **低角度阳光穿过长大气路径被瑞利散射滤成红橙光,再从下方照亮高/中层云的云底**。
它不是"有云 + 有太阳"就行,而是一套**光路几何 + 云幕条件 + 大气洁净度**三者同时满足的耦合事件。任一硬条件不满足 → 直接不烧,与其他条件多好无关(这是**乘法/门槛**结构,不是加权求和)。

---

## 1. 物理机理

1. 日出/日落时太阳接近或低于地平线,阳光在大气中的**斜穿路径极长**。
2. 瑞利散射 ∝ λ⁻⁴:蓝紫光被大量散射掉,只剩**红橙黄**长波能穿透到远处。
3. 这束被"染红"的光**从下方**打到云的底面 → 云底被染成红/金/粉。
4. 太阳**位于地平线以下**时(日落后 civil twilight 段),直射光已滤成以红光为主 → 颜色偏**粉红**,观赏效果最佳,这才是严格意义的"火烧云";太阳还在地平线之上时多是金黄"霞光"。
5. "分界线"现象:地球阴影(terminator)在云幕上投出一条锐利明暗界线,界线之上被照亮、之下入影——界线高度由云高与太阳俯角几何决定(CMA 科普)。

---

## 2. 必要条件(硬门槛,不满足即否决)

来自中国气象局 + sunsethue + Corfidi 三方一致的"观赏三要件":

| 门槛 | 内容 | 不满足的后果 |
|---|---|---|
| **A. 透光通道** | 太阳方位角一侧(晚霞=**西**,朝霞=**东**)近地平线**有一条无云/无降水的缝隙**,让低角度光能无遮挡射到云幕 | 通道被低云/云带/降水堵死 → 光到不了云幕 → **即使头顶云再好也不烧**。这是最常见的"预报好却没烧"主因 |
| **B. 有云幕** | 观测地上空有**中/高层云**作"幕布"接光 | 万里无云 → 没有承接体 → 天空只有渐变,无火烧云 |
| **C. 本地大气通透** | 观测地能见度高、无雾/霾/沙尘、湿度不过高 | 雾霾把颜色"洗淡"成灰粉/苍白(Corfidi:气溶胶**subdue**而非增强);低云盖顶则直接遮住云幕 |

> 关键推论:**通道(A)与云幕(B)在空间上是分离的**——通道在"太阳方向、远处、近地平线",云幕在"观测地上空"。二者要分开评估,不能用一个"总云量"标量概括(现有模型的致命简化)。

## 3. 质量因子(满足门槛后,决定烧得多壮观)

### 3.1 云高(第一质量因子)
- **低云 <2000m**:基本**不利**。云底低、太阳光被地平线/边界层遮挡,即便点亮也短暂寡淡;更常见是盖顶遮挡云幕。例外:边界层异常洁净(热带海面)时低云也能上色。
- **中云 2000–6000m(高积云/高层云)**:**优质幕布**。反射时间长,是壮观火烧云的主力,"鱼鳞/波浪"结构最出片。
- **高云 >6000m(卷云/卷积云)**:**最佳色彩**。截获未经边界层污染的"纯净"阳光,日落后仍能被照亮长达 ~30 分钟,颜色最干净通透。
- 经验反差:**质量最高的大烧云底往往偏低 → 更炽烈但持续更短(<20 分钟)**;高云主导则更持久、更"高级灰"通透。二者取向不同,不是越高越好一条道。

### 3.2 云量(倒 U 形,非单调)
- 最佳**总云量 40–60%**(sunsethue)/ 40–70%(社区经验)。
- 太少(<15–20%):没幕布。
- 太多/满盖(>80–90%):糊死一片、无破口、无层次、通道也常被堵 → 差。
- **要点:同样"80% 高云",带破口透光的 vs 均匀糊死的,结果天差地别——标量云量分不出,必须看空间形态(云图)。**

### 3.3 云型(纹理/结构越丰富越好)
sunsethue "Very Good" 档:**高积云(altocumulus)、卷积云(cirrocumulus)、卷云(cirrus)**。
- 好:鱼鳞、波浪、絮状、条带、有破口有层次的中高云。
- 差:均匀层云/层积云铺满(stratus/stratocumulus 盖顶)、积雨云砧板遮挡。

### 3.4 大气洁净度 / 气溶胶(Corfidi 核心论点)
- **清洁空气是所有绚烂日落的共同主料。** 气溶胶(0.5–1μm 污染液滴)做非选择性散射,把红橙**冲淡**成苍白黄粉。
- 适度尘埃可"加深"红调(折射),但整体上**洁净 > 浑浊**;雾霾/高 AOD 是减分项。
- 季节:秋冬空气环流弱、霾产出低 → 常有更好的霞(北方尤其)。

### 3.5 湿度(第二大气因子)—— ⚠️ 区分"地表 RH"与"高空各层 RH",两者用途相反
- **地表/边界层 RH(用于判大气洁净度)**:最佳 **40–60%**;**>85% 明显变差**(近饱和成雾/低云,颜色被压制)。地表 RH 低 = 边界层干净通透,是好事。
- **高空各气压层 RH(用于反演"哪个高度有云")**:美国专利 US20170109634A1 的做法——用各气压层 RH 反推该层是否成云:
  - 高云层 300–200mb:**RH 50–70% → 有高云**
  - 中云层 500–700mb:**RH 60–80% → 有中云**
  - 低云层 925–800mb:**RH 80–90% → 有低云**;**>90% → 视为该层满盖**
  - 用途:Open-Meteo 提供各气压层 RH,可在点预报缺直接云量时**据此反演分层云**,或与模式云量互校。**注意:高空 RH 高=有云(可能是好画布),地表 RH 高=坏——方向相反,建模时务必分开。**

### 3.7 气压变化 / 天气系统趋势(SunsetWx 强调,原框架漏掉)
- SunsetWx 约 20 因子里,**气压变化**是核心之一:锋面/天气系统过境的**趋势**决定通道是"正在打开(雨后转晴,气压回升)"还是"正在关闭(云系逼近,气压下降)"。
- 与 §3.6 的"雨后初晴 vs 正在下"、§4 的"晚霞=西边已晴"一脉相承:**单帧云量不够,要看气压/云系的时间趋势**(→ 卫星连帧动画 + 多时次预报趋势)。

### 3.6 降水(否决级)
- 光路上或本地正在**降水 → 否决**。
- 但**雨后初晴(雨刚过、云正在裂开)是出大烧的经典时机**(北京多次雨后火烧云案例)——因为通道刚打开、空气被雨洗净、天上仍留破碎中高云幕。**"正在下"与"刚下完"是相反信号**,时间窗判断很关键。

## 4. 朝霞 vs 晚霞(方向 + 天气含义)

"朝霞不出门,晚霞行千里"的科学解:
- 天气系统在中纬度**自西向东**移动。
- **晚霞**:被照亮说明**西边已放晴无云**(通道在西)→ 未来天晴 → "行千里"。晚霞是**好通道 + 好天**的信号。
- **朝霞**:通道在东、光从东来;而朝霞出现往往说明**西边正有云系/水汽逼近**(东边暂晴让光透过,西边云在压过来)→ 预示转雨 → "不出门"。
- 对预测的含义:
  - 晚霞评通道看**西侧**方位角剖面;朝霞看**东侧**。
  - 朝霞的"存档边界/单模式"问题外,还要注意朝霞常伴随天气转折,假阳性/真阳性都可能更多变。

## 5. 卫星云图判读指南(可见光 + 红外)

核心:**可见光看厚度与纹理(判云型),红外看云顶温度即高度(判云高)。两者配合才能定云。**

| 云型 | 高度 | 可见光(truecolor) | 红外(BT,越冷越亮/白) | 对火烧云 |
|---|---|---|---|---|
| 卷云 Ci | 高 >6km | 灰、纤维状细丝、半透明(薄→不太反光) | **很亮/很冷**(高) | 好·纯净色彩·持久 |
| 卷积云 Cc | 高 | 细密鱼鳞小块 | 亮/冷 | 很好 |
| 高积云 Ac | 中 2–6km | 中等亮、块状/波浪/鱼鳞、有破口 | 中等灰(中温) | **最佳幕布** |
| 高层云 As | 中 | 灰白成片、较均匀 | 中等 | 一般(易糊) |
| 层积云 Sc | 低 <2km | 较亮、**胞状**有缝隙的块 | **暗/暖**(低) | 差(易盖顶/堵通道) |
| 层云 St | 低 | 均匀铺满、边界平滑 | 暗/暖 | 差 |
| 积云 Cu | 低(顶可高) | 亮、离散团块、有影 | 顶较暖(除非发展旺) | 通道处出现=堵 |
| 积雨云 Cb | 贯穿 | 极亮、砧状 | 极冷极亮、砧板扩展 | 遮挡/降水,否决级 |

判读要点:
1. **判高度用红外亮温**:同样一片白,红外很亮=高云(冷顶),红外发暗=低云(暖顶)。这是区分"高云画布"和"低云盖顶"的关键——正是标量云量分不出、必须看图的地方。
2. **判通道看太阳方位角一侧近地平线的剖面**:晚霞看观测点向西约 **100–400km** 走廊上是否有低云带/降水(离太阳越近的低云越致命,因为它最靠近光源方向堵光)。
3. **判破口/层次**:云幕区要有缝隙、有纹理、非均匀糊死。均匀满盖(可见光大片纯白、红外均匀)= 差。
4. **动画看趋势**:峰值时刻前 30–60 分钟连帧,看云是在**裂开(转好)**还是**压过来(转差)**——雨后裂云 vs 云系逼近,信号相反。
5. 夜间/弱光:可见光不可用时以**红外为主**判高度与结构(火烧云发生在晨昏,可见光光照角低,红外更稳)。

## 6. 量化框架 v2(综合 sunsetbot / 五因子 / sunsethue)

三家方法论的共识可归纳为**"门槛 × 云幕质量 × 大气修正",而非线性加权**:

```
概率/质量  ≈   [通道门槛 G_channel]           # 0 或 ~1,西/东走廊近地平线是否通透
             × [云幕质量 Q_canvas]            # 0–10,中高云的"量×型×高×破口"综合
             × [大气清洁修正 M_air]           # 0–1,能见度/AOD/湿度
             × [降水/雾霾否决 G_hard]          # 0 或 1
```

- **G_channel(门槛)**:晚霞看西、朝霞看东,沿太阳方位角 100–400km 采样低云/总云;被堵超阈值→趋零。**必须方向性、空间性**,不能用本地总云量替。参考专利 US20170109634A1 的**扇区法**:把观测点周围切扇区、逐扇区评估,并**对太阳对侧扇区降权**(对侧被戏剧性光照亮概率低);扇区半径按地形/能见距离可调。
- **气压/趋势项(可作 G_channel 与 M_air 的时间修正)**:SunsetWx 用气压变化判天气系统趋势——通道"正在打开(雨后转晴)"给正修正,"正在关闭(云系逼近)"给负修正。落地上用**卫星连帧动画 + 多时次预报**判裂/压趋势(§3.6/§3.7/§4)。
- **Q_canvas(核心质量,0–10)**:不是单看总云量,而是**中高云的覆盖(40–70% 倒U)× 云型加成(Ac/Cc/Ci 高)× 云高加成 × 是否有破口层次**。理想上应由**云图空间形态**给出(→ 这正是要接卫星云图 + 多模态判读的原因),标量点预报只能给粗代理。
- **M_air(0–1)**:AOD/能见度/RH。清洁=1,雾霾/高湿逐级降。缺数据不惩罚(置信度另算)。
- **G_hard**:光路或本地降水、浓雾 → 0。注意区分"正在降水"(否决)与"雨后初晴"(利好)。

**概率 vs 质量分离**:
- **质量分**=上式的期望强度(能烧多好)。
- **概率/置信度**=多模式(EC/GFS/ICON/CMA)对上述条件的**一致性**;分歧大→置信度低(sunsetbot 用混淆矩阵表达"预报某档时,实际各档的概率分布")。

**数据源分工**:
- 点/剖面预报(Open-Meteo 多模式,免费)→ G_channel、云量、湿度、降水、AOD 的**预测代理**。
- 卫星云图(Himawari,可见光+红外)→ 实况的**云型/云高/破口/通道空间形态**,是 Q_canvas 的真信号来源与临近修正。
- 多模态判读(看云图 + 相似历史案例 RAG)→ 把空间形态翻译成质量修正。

## 7. 对现有模型的批判(连接 ρ=0.026)

现规则模型 `fire_cloud_score = canvas × channel × local_low × aerosol × precip`,首测 Spearman ρ=0.026(近零)。对照本文档,病因:

1. **通道因子空数据 → 恒 1.0**:未拉到方位角剖面(§2-A 的硬门槛完全失效)。四个乘子里三个(通道/气溶胶/降水)在回填数据里恒 1.0,模型退化成**只由 canvas 单因子驱动**。
2. **canvas 用标量 cloud_high,且"100%→0"过猛**:把"满天高云"一律判 0,分不清"糊死"与"带破口",违反 §3.2/§5.3——正是"必须看云图"的地方用了标量。多条实际大烧(cloud_high=100)被打 0。
3. **多模式对某历史小时点云量分歧极大,均值搅糊**(§6 应改为"一致性→置信度",而非把分歧平均进质量分)。
4. **无云高/云型区分**:point forecast 的 high/mid/low 是覆盖率不是"云型/破口",丢掉了 §3.3、§5 的全部空间信息。

→ 结论与用户判断一致:**标量五因子只能当粗筛;真正的质量信号在卫星云图空间形态**。下一步价值排序:(a) 通道因子喂真实方位角剖面数据(救活 §2-A);(b) 接卫星云图(可见光+红外)作 Q_canvas 真信号与临近修正;(c) 多模态判读 + 经验库把"这张图为什么 X 分"沉淀成 RAG 案例。

## 8. 经验库(RAG)案例笔记应记什么

每个打分日,除 date/event/score,应沉淀:
- **通道**:西/东走廊 100–400km 当时低云/降水情况(通/堵)。
- **云幕**:主导云型(Ac/Cc/Ci/Sc…)、云高档、覆盖率、有无破口层次。
- **大气**:能见度/AOD/RH 档、是否雨后初晴。
- **卫星形态描述**:可见光纹理 + 红外高度判读的一句话(如"红外见高积云带自西裂开,西侧通道 200km 无低云")。
- **为什么是这个分**:一句因果结论(如"通道通+高积云带破口+空气洁净→满分";或"头顶高云满但西侧 150km 低云带堵通道→预报高实际 2 分,典型假阳性")。
- 可附卫星帧路径 + 现场照片。
